    TEnmo Money Transfer App

    "This project was one of our capstone group projects at Tech Elevator. We were tasked with building a money transfer application, much like Venmo, that operates from the command line. Our API needed to be RESTful and we wrote the server and client in java, following the MVC design pattern. On the server side we had DAOs to access the database and create objects with our model classes. Our controller handled incoming requests from the client and required authentication. Our server made use of the Spring Framework and dependency injection. On the client side we created a service class that sent requests to our server containing JWTs for authenitcation. Our console service class was responsible for the view and formatted displays for the user."

    "After we completed and handed in the assignement, I decided I wanted to redo it to add in some additional concepts we learned. Our initial project had grouped too many methods into one DAO class that did not relate to the same model or table. So I created more model classes and additional DAOs. This resulted in more organized and understandable structure. Additionally, I added a service layer that interacted with the DAOs and handled the business logic. This meant the controller only needed to access the service layer to access the necessary information, and if the project grows it will be more easily scaleable. On the client side I removed some of the logic from our console service that felt unrelated to the display. I put that logic in its own class. I also broke up some of the method into smalled reusable methods to the DRY rule. Lastly, my group ran out of time and couldn't implement the optional use cases, so I added functionality for those use cases as well."

    "I plan to continue working on this project when I can. I suspect the client side can be cleaned up even further. I would also like to add transactions to the server, add in customer exceptions, and write unit tests for the code."
